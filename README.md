# ALerts

A curated feed of current AI news for i2e Consulting — a Proof of Concept from AI Labs.

**Live site:** https://ak-1096.github.io/i2e-News/

ALerts (formerly NewsPulse AI) raises AI awareness across the organisation with a clean, human-curated feed of AI
developments. It uses a **split-surface** architecture: a curator publishes from inside Teams, and
everyone reads from a simple static site.

## Architecture

Two surfaces, joined by a single data contract — they never call each other directly.

```
  Curator Agent (write)                 data/articles.json                 Static Reader (read)
  Copilot Studio in Teams   ───writes──▶   the data contract   ───reads──▶  GitHub Pages site
  RSS + Bing · summarise                (id, title, url, …)                 list · archive · article
```

- **Curator Agent** — a Microsoft Copilot Studio agent in Teams (the *write* path). It finds AI news
  (curated RSS + Bing), lets a curator select and summarise articles, writes them to
  `data/articles.json`, and pings the Teams group with a link. **This lives in the M365 tenant, not
  in this repo** — see the build guide: [`docs/curator-agent-runbook.md`](docs/curator-agent-runbook.md).
- **Static Reader** — this repository (the *read* path). A no-backend static site that renders the
  list, archive, and per-article pages from `data/articles.json`.

## This repository (the static reader)

| Path | Purpose |
|------|---------|
| `index.html` | Front page — the most recent articles (capped, see below). |
| `archive.html` | Full published history, newest first. |
| `article.html` | Per-article view, reached by `?id=<id>` — the Teams-ping target. |
| `assets/app.js` | Shared logic: loads/sorts `articles.json`, renders rows, `RECENT_LIMIT`. |
| `assets/styles.css` | Shared styling. |
| `data/articles.json` | **The data contract** — the site's only data source. |
| `data/articles.schema.json` | JSON Schema for the contract, enforced in CI. |
| `.github/workflows/pages.yml` | Validates `articles.json`, then deploys to GitHub Pages. |
| `docs/curator-agent-runbook.md` | How to build the Copilot Studio curator agent. |

The front page shows the most recent **`RECENT_LIMIT`** items (default **10**, set in
`assets/app.js`); everything else is reachable via the archive.

## Data contract — `data/articles.json`

An array of article objects. The curator agent writes it; the static site reads it. Every object has
all eight fields (validated against `data/articles.schema.json`):

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Stable unique id; used in the article page URL (`article.html?id=…`). |
| `title` | string | Article headline. |
| `url` | string | Original source URL (link-out target), `http(s)://…`. |
| `source` | string | Publication name. |
| `summary` | string | Short summary generated by the agent at publish time. |
| `topic` | string | `"Latest"` or the topic prompt used to find the article. |
| `publishedDate` | string | Original publication date, `YYYY-MM-DD`. |
| `addedDate` | string | When the curator published it; drives ordering, `YYYY-MM-DD`. |

## Local development

The pages fetch `data/articles.json` over HTTP, so serve the folder (don't open the files via
`file://`):

```bash
python -m http.server 8000
# then open http://localhost:8000/
```

Validate the data contract against the schema (the same check CI runs):

```bash
npx ajv-cli@5 validate -s data/articles.schema.json -d data/articles.json
```

## Deploy

Pushing to `main` runs [`.github/workflows/pages.yml`](.github/workflows/pages.yml): a `validate`
job checks `data/articles.json` against the schema, and only if it passes does the site deploy to
GitHub Pages. A malformed write therefore fails the build instead of corrupting the live archive.

## Scope

This is a Proof of Concept governed by the BRD's PoC Fence. **Out of scope** for the static reader:
search, filtering, tags, reactions, analytics, access gating, and any AI processing on the site
itself. Those are Phase 2 / change-request items.
